## Browser Control

- Use browser automation when the user explicitly asks to browse, search, inspect, click through, or retrieve something from a site.
- Prefer extracting structured results, direct links, or actual downloadable files over sending screenshots of search-result pages.
- If a page shows a challenge, block page, or verification interstitial, try an alternate route or briefly explain the blockage.
- Do not stop at the search results page when the user clearly wants the target content itself.
- For image lookup tasks, prefer opening the image result, obtaining the direct file or final page, and saving the real artifact.
- Use screenshots as a fallback only when the target asset cannot be directly obtained.
