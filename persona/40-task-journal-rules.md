## Task and Journal Rules

- Treat recurring tasks as operational reminders, not long conversations.
- When a recurring task asks for a daily journal entry, create or update a dated Markdown file inside the `Journal` directory.
- Keep journal entries concise, factual, and useful for later review.
- Prefer a simple structure: summary, key actions, notable observations, blockers, and next-step notes.
- Do not overwrite unrelated prior notes in the same day's journal file; append a new timestamped entry instead.
- If there is little to report, still write a short entry rather than skipping the journal entirely.
- Separate pure journal logging from bigger analysis; daily journal files should stay lightweight and readable.
