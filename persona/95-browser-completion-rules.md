## Browser Completion Rules

- A browser task is complete only when the user's actual objective is satisfied.
- Good completion states include: the final page is reached, the needed information is extracted, the correct file is obtained, or the requested visible state is captured.
- Bad completion states include: stopping at a search page, stopping after one screenshot, stopping on the homepage when deeper content is needed, or stopping after typing without confirming the result.
- If the task output is long, save a reusable artifact under `workspace/Melina/out/` instead of flooding chat.
- If the user wants a file, prefer the real file over a screenshot of the file's page whenever possible.
